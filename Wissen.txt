🌱 GitHub Schritt-für-Schritt für Einsteiger


---

🧩 Schritt 1: Konto erstellen

1. Öffne https://github.com


2. Klicke oben rechts auf Sign up


3. Gib deine E-Mail-Adresse ein, wähle ein Passwort und einen Benutzernamen.


4. GitHub fragt evtl. nach einer Sicherheitsprüfung (z. B. kleine Rätsel).


5. Danach bekommst du eine Bestätigungs-E-Mail – öffne sie und klick auf den Link.



→ Jetzt hast du dein GitHub-Konto! 🎉


---

📦 Schritt 2: Dein erstes Projekt (Repository) erstellen

1. Wenn du eingeloggt bist, siehst du oben rechts ein + → klicke darauf → New repository


2. Wähle:

Repository name: z. B. mein-erstes-projekt

Description (optional): z. B. „Mein erstes GitHub-Testprojekt“

Public (damit es sichtbar ist)

✅ „Add a README file“ (wichtig! Sonst ist das Projekt leer)



3. Klicke Create repository



→ Jetzt hast du dein eigenes GitHub-Projekt – wie ein leerer Ordner mit einer kleinen Beschreibung.


---

📄 Schritt 3: Verstehen, was du siehst

Dein Repository hat nun z. B. so eine Ansicht:

mein-erstes-projekt
│
├── README.md   ← eine Textdatei mit Infos über dein Projekt

Die Datei README.md ist eine Art Startseite oder Beschreibung deines Projekts.

Du kannst sie direkt im Browser bearbeiten.



---

✏️ Schritt 4: README-Datei bearbeiten

1. Klicke auf README.md


2. Klicke oben rechts auf das ✏️ Symbol („Edit this file“)


3. Schreibe etwas hinein, z. B.:

# Mein erstes Projekt 🎉
Dies ist mein erster Test auf GitHub.
Ich lerne gerade, wie man Repositories benutzt.


4. Scrolle runter und schreibe bei Commit changes z. B.:

Update README mit Begrüssung

Dann klicke Commit changes ✅



→ Du hast gerade deine erste Änderung (Commit) gemacht! 🎊
GitHub hat diese Änderung gespeichert und merkt sich jede Version.


---

📂 Schritt 5: Eine neue Datei hinzufügen

1. Klicke oben auf Add file → Create new file


2. Nenne sie z. B. ideen.txt


3. Schreib etwas rein, z. B.:

💡 Ideen für mein erstes GitHub-Projekt:
- Eine kleine Webseite
- Ein Kochbuch
- Eine Liste mit Lieblingsfilmen


4. Unten wieder auf Commit new file klicken.



→ Jetzt hast du zwei Dateien in deinem Projekt.
Wenn du auf die Startseite klickst, siehst du beide.


---

🕰️ Schritt 6: Alte Versionen ansehen

Jede Änderung („Commit“) ist gespeichert.
Wenn du auf History klickst, siehst du:

wann du etwas geändert hast,

was genau sich verändert hat,

und du kannst jede alte Version wiederherstellen.


So kannst du nie wieder „aus Versehen etwas löschen“, ohne es zurückholen zu können.


---

👥 Schritt 7: Mit anderen zusammenarbeiten

Wenn du z. B. einem Freund dein Projekt zeigen willst:

Schicke einfach den Link, z. B.
👉 https://github.com/deinName/mein-erstes-projekt


Wenn du möchtest, dass jemand mitarbeitet:

Gehe auf Settings → Collaborators

Lade eine Person ein, z. B. per GitHub-Benutzername oder E-Mail


Beide können dann Änderungen vorschlagen oder Dateien hochladen.


---

🎨 Schritt 8: Beispielideen für Projekte (auch ohne Programmieren)

Idee	Beschreibung

📖 Notizbuch oder Tagebuch	Eine Sammlung deiner Ideen, Gedanken oder Lernfortschritte
🍳 Rezeptbuch	Lade eigene Rezepte hoch, und andere können sie erweitern
🎵 Playlist-Sammlung	Liste deiner Lieblingssongs, nach Stimmung sortiert
🧘 Motivationszitate	Tägliche Zitate oder Gedanken, die dir gefallen
📚 Lernprojekt	Sammle alles, was du über KI, GitHub oder andere Themen lernst


👉 Du kannst GitHub also wie eine öffentliche Bibliothek oder Lernjournal benutzen.


---

🧠 Schritt 9: Wenn du später programmierst

GitHub wird dann dein bester Freund:

Du kannst dort Code speichern

mit anderen Entwicklern zusammenarbeiten

und automatisch testen, ob dein Code funktioniert.


Beispiel: Ein Python-Skript auf GitHub

print("Hallo Welt!")

Später kannst du daraus ganze Webseiten, Apps oder KI-Modelle bauen.


---

🚀 Schritt 10: Bonus – GitHub Pages (deine eigene kleine Webseite!)

GitHub kann automatisch aus deinen Dateien eine Webseite machen!

1. Gehe in dein Repository → Settings → Pages


2. Wähle als Quelle main branch


3. Speichern
→ Nach ein paar Minuten bekommst du eine URL wie:



https://deinName.github.io/mein-erstes-projekt/

So kannst du ohne Programmierkenntnisse deine eigene kleine Website haben – gratis.

📲 Verfügbar für:

iPhone / iPad → im App Store

---

🌟 Was du mit der GitHub-App machen kannst

Die App ist vor allem da, um Projekte unterwegs zu verwalten, nicht um viel Code zu schreiben.
Sie ist also eher für Organisation, Kommunikation und Übersicht gedacht.

Funktion	Beschreibung

👀 Projekte durchsehen	Du kannst deine Repositories anschauen, Dateien öffnen und den Code ansehen.
📝 Kleine Änderungen machen	Du kannst Texte oder kleine Codezeilen direkt im Handy bearbeiten.
💬 Diskutieren / kommentieren	Du kannst Kommentare schreiben, auf Issues oder Pull Requests antworten.
🔔 Benachrichtigungen	Du siehst, wenn jemand an deinem Projekt arbeitet oder dich erwähnt hat.
🌍 Neue Projekte entdecken	Du kannst andere Repositories durchsuchen, folgen oder „starren“ (liken).


📌 Beispiel:
Du bekommst unterwegs eine Nachricht:

> „Kannst du bitte die Beschreibung im README anpassen?“
→ Öffne die App → Datei antippen → Text ändern → „Commit changes“.
Fertig – kein Laptop nötig!

💻 2. GitHub Desktop (für Windows & Mac)

Für Computer gibt’s zusätzlich die GitHub Desktop App:
👉 https://desktop.github.com/

💡 Wozu sie gut ist:

Sie ist perfekt, wenn du lokal (auf deinem PC) an Dateien arbeitest und sie einfach mit GitHub synchronisieren willst.

Funktion	Beschreibung

🖥️ Repositories herunterladen (clone)	Du lädst dein Online-Projekt auf deinen PC.
💾 Änderungen speichern (commit)	Du speicherst neue Versionen direkt lokal.
☁️ Hochladen (push)	Du lädst deine neuen Versionen auf GitHub hoch.
🔄 Aktualisieren (pull)	Du holst dir die neuesten Änderungen von anderen.
👀 Vergleiche ansehen	Zeigt dir, was sich geändert hat.


Beispiel:
Du schreibst zu Hause an einer Datei → speicherst sie → klickst „Push to GitHub“ →
und sie ist online auf deiner GitHub-Seite.

🌱 Schritt 1: Repository erstellen oder auswählen

Du brauchst ein Repository (ein Projekt auf GitHub), das deine Webseite enthalten soll.
Das kann entweder:

dein bestehendes Projekt sein, oder

du machst ein neues (z. B. mein-webprojekt)


So erstellst du eins:

1. Gehe auf https://github.com/new


2. Name: mein-webprojekt


3. Öffentlich (Public)


4. ✅ Häkchen bei “Add a README file”


5. Klicke Create repository




---

📄 Schritt 2: Eine einfache Webseite-Datei hinzufügen

Du brauchst mindestens eine HTML-Datei, die GitHub anzeigen kann.

1. Klicke oben auf Add file → Create new file


2. Nenne sie: index.html


3. Füge folgenden Inhalt ein (das ist eine einfache Webseite):

<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>Willkommen auf meiner GitHub Page</title>
  <style>
    body {
      font-family: sans-serif;
      background-color: #f2f2f2;
      text-align: center;
      padding-top: 100px;
    }
  </style>
</head>
<body>
  <h1>👋 Hallo Welt!</h1>
  <p>Dies ist meine erste GitHub-Webseite 🎉</p>
</body>
</html>


4. Scrolle nach unten und klicke auf Commit new file




---

⚙️ Schritt 3: GitHub Pages aktivieren

1. Klicke oben im Repository auf Settings
(bei Handy: Menü öffnen → „Settings“)


2. Scrolle ganz nach unten zum Bereich “Pages”


3. Dort steht:

> “Build and deployment”




4. Wähle bei „Source“:
→ Deploy from a branch


5. Dann bei „Branch“:
→ main
→ und / (root) (das bedeutet: die Hauptordner-Dateien)


6. Klicke auf Save




---

⏳ Schritt 4: Warten (1–2 Minuten)

GitHub braucht kurz Zeit, um deine Seite zu „bauen“.
Nach 30 Sekunden bis 2 Minuten erscheint oben eine Nachricht:

> ✅ Your site is live at
https://deinName.github.io/mein-webprojekt/



Wenn du draufklickst → tadaaa 🎉 — deine Webseite ist online!


---

🧩 Schritt 5: Änderungen machen

Wenn du die Webseite ändern willst:

1. Gehe auf deine Datei index.html


2. Klicke auf ✏️ (Edit)


3. Ändere den Text oder Stil


4. Scrolle runter → Commit changes



→ GitHub aktualisiert automatisch deine Webseite in wenigen Sekunden.
